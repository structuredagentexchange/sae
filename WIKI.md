## What SAE is

`SAE` means `Structured Agent Exchange`.

It is a small, neutral way to describe the shape of agent work:

- objective
- constraints
- deliverable
- delegation hints
- review posture
- optional preference and instruction hints

## What SAE is not

SAE is not:

- a transport protocol
- a replacement for `A2A`
- a full agent runtime
- a requirement to adopt any single internal dialect
